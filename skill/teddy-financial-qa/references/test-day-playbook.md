# 测试日执行手册

## 1. 目标

在测试数据公布后，尽快、稳定地产出符合要求的结果文件，而不是临场改架构。

## 2. 测试日前必须具备

- 固定项目环境
- 固定输入输出目录
- 固定批处理入口
- 固定导出逻辑
- 固定检查清单
- 至少一次完整模拟测试

## 3. 测试日处理模式

### 模式 A：只公布新问题
直接基于现有数据库和知识库运行任务二、任务三导出流程。

### 模式 B：公布新问题和新文档
先执行增量解析、入库和索引，再运行任务二、任务三导出流程。

## 4. 检查清单

- Excel 文件名是否正确
- 图表是否保存到 `result/`
- 图片命名是否符合题号约定
- JSON 结构是否完整
- `references` 是否缺失
- 是否存在空答案或异常值
- SQL 或检索结果是否明显不合理

## 5. 时间分配建议

- 前 30 分钟：读取测试说明，确认数据结构和提交格式
- 中段：执行批处理与人工抽查
- 最后至少预留 30–60 分钟检查并上传

## 6. 禁止事项

- 测试当天升级依赖
- 临时大改目录
- 手工逐题补丁式修复为主流程
- 把输出散落在多个地方
