# 演示与答辩建议

## 1. 演示定位

把系统呈现为“智能问数工作台”，而不是普通聊天机器人。

## 2. 推荐页面结构

- 系统总览
- 财报解析入库
- 智能问数
- 研报增强与归因

## 3. 演示顺序

1. 系统总览：讲清数据底座 + 问数链路 + RAG
2. 任务一：展示 PDF、抽取结果、标准化、校验
3. 任务二：展示问题、SQL、图表、JSON、Excel
4. 多轮澄清：展示缺失条件时的追问
5. 任务三：展示多意图拆解、references 和归因结果

## 4. 评委最看重的点

- 数据是否可信
- 查询链路是否清晰
- 输出是否稳定
- 结果是否可解释
- 是否满足比赛文件输出要求

## 5. 不推荐的展示方式

- 只放一个聊天框
- 把 SQL 和来源都藏起来
- 现场从零重新解析所有数据
- 把大量精力花在炫技前端上
